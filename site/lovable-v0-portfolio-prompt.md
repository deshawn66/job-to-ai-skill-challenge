# Lovable Or v0 Portfolio Site Prompt

Build a polished portfolio site for a public challenge called "The Job-To-AI Skill Challenge."

Audience: business operators, founders, sales leaders, and professionals who are AI-curious but not developers.

Tone: direct, useful, plain English, no hype.

Primary headline:

I Turned My Job Into 12 AI Skills

Primary goal:

Show a public portfolio of 12 reusable AI skills created from one real workday.

Site structure:

1. Hero section with the headline, short subheadline, and two buttons:
   - View The Skills
   - Open The GitHub Repo
2. Short explanation section:
   - Most AI advice starts with tools.
   - This project starts with the job.
   - I tracked one real workday and turned repeatable work into 12 reusable skills.
3. Agent grid with 12 cards.
4. Challenge steps section.
5. Proof of completion section.
6. Final call to action.

Design direction:

- Professional operator portfolio.
- Clean, modern, and credible.
- Avoid crypto, sci-fi, or generic AI visual language.
- Use a practical workspace feel: notes, task cards, dashboard fragments, and repo links.
- Make it easy to scan.
- Each agent card should have a distinct accent color.
- Mobile-friendly.

Agent card content:

Each card should include:

- Agent name
- Job pain solved
- Tools it may touch
- What the user gives it
- What it produces
- Safety boundary
- GitHub link

The 12 agents:

1. Daily Work Capture
2. Repeat Task Finder
3. Meeting To Action Plan
4. Inbox Follow-Up Drafter
5. Tool Workflow Brief
6. Scorecard Explainer
7. Coaching Prep
8. Playbook Builder
9. Compliance First Pass
10. Founder Research And Feedback
11. LinkedIn Build-In-Public Post
12. Scary Tool Review

Challenge steps:

1. Track one real workday.
2. Find the repeatable tasks.
3. Turn 12 tasks into AI skills.
4. Publish the repo.
5. Build the portfolio.
6. Post the launch.
7. Research AI tools in your field.
8. Message founders with real feedback.
9. Try one scary tool.
10. Turn one course lesson into another skill.

Final call to action:

Start with the work you already repeat. That is where the best automation ideas are hiding.
