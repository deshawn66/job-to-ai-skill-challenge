# GitHub Publishing Checklist

GitHub is the public filing cabinet for the challenge. Use it to show the work.

## Before Publishing

- Remove customer names.
- Remove borrower, patient, client, or employee details.
- Remove passwords and API keys.
- Remove screenshots with personal information.
- Replace private company examples with public-safe examples.
- Make sure every skill explains what it should never do.

## Recommended Repo Settings

- Visibility: Public
- Repo name: `job-to-ai-skill-challenge`
- Description: `Turn one real workday into 12 reusable AI skills.`
- License: MIT or Creative Commons Attribution 4.0
