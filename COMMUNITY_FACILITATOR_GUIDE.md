# Community Facilitator Guide

Use this to run the challenge with a group.

## Positioning

This challenge is for people who know their work but do not know how to turn that knowledge into AI leverage.

The core message:

> You do not need to become a developer first. Start by documenting the work you already repeat.

## Session Format

Run this as a 4-week sprint.

### Week 1: Workday Capture

Goal: Capture the real job, not the polished job description.

Group prompt:

> What did you actually do yesterday that you have done before and will probably do again?

Deliverable:

- Completed day tracker worksheet.
- 20 to 40 raw task entries.
- 12 candidate skills selected.

### Week 2: Skill Build

Goal: Convert repeatable work into reusable instructions.

Group prompt:

> If a smart assistant watched you do this task five times, what would it need to remember?

Deliverable:

- 12 skill folders.
- Each skill has a README, sample input, sample output, and safety notes.

### Week 3: Portfolio Site

Goal: Make the work understandable to outsiders.

Group prompt:

> If someone saw your job portfolio, would they understand what you built and why it matters?

Deliverable:

- Portfolio site draft.
- 12 agent cards.
- GitHub links connected.

### Week 4: Public Point Of View

Goal: Move from private tinkering to public judgment.

Group prompt:

> What do you now believe about AI in your field that you did not believe four weeks ago?

Deliverable:

- LinkedIn launch post.
- Startup scorecard.
- Founder message sent.
- Scary-tool review.
- One course lesson converted into a skill.

## Review Questions

Use these to push the group beyond surface-level automation:

- Is this a real task or a made-up portfolio task?
- Would this save time every week?
- Does the skill need private data, or can it work from sanitized examples?
- Does the sample output look like something you would actually use?
- What should the AI never do on this task?
- What human judgment still belongs to you?

## Completion Standard

The work is complete when a stranger can open the repo, understand the job, copy a skill, and know how to use it safely.
