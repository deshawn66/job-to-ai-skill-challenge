# Scary Tool Review

Turn an intimidating tool experiment into an honest field report.

## Inputs

- Tool name
- Why the user tried it
- Setup notes
- Tasks attempted
- What worked
- What confused the user
- What felt powerful
- What still feels risky

## Workflow

1. Explain why the tool was worth trying.
2. Separate setup friction from product value.
3. List what worked and what failed.
4. Explain what felt powerful in plain English.
5. Name what still feels too early.
6. Give a recommendation for people in the user's role.

## Output

Return:

- Field report
- What confused me
- What surprised me
- What worked
- What failed
- What feels powerful
- What still feels too early
- Recommendation

## Safety

- Do not connect private systems casually.
- Do not advise running unknown tools on sensitive files.
- Do not pretend early tools are production-ready.
- Make uncertainty visible.
