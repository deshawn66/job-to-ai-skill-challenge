# The Job-To-AI Skill Challenge

Turn the job you already do into a public AI portfolio.

This is not an "AI 101" challenge. The point is to document how you actually spend your day, turn repeatable work into reusable AI skills, publish the useful ones, and build a public point of view.

Core prompt:

> This was my job and how I spent my day. How can you help me automate it with skills?

A skill is a reusable instruction set for AI. Think of it like a playbook that teaches an AI assistant how to do one repeatable task in your world.

## Who This Is For

- Operators
- Founders
- Sales leaders
- Recruiters
- Consultants
- Mortgage professionals
- Anyone who knows their work deeply but does not write code

## The Challenge

1. Track one real workday.
2. Turn the repeatable parts into 12 AI skills.
3. Publish the 12 skills to GitHub.
4. Build a portfolio site with one agent card per skill.
5. Post the site and GitHub repo on LinkedIn.
6. Search "AI for my job" and try the new tools.
7. Message founders with useful feedback.
8. Try one tool that feels intimidating.
9. Take one tactical AI course in your field.
10. Build, share, learn, improve, repeat.

## What You Will Build

- A public GitHub repo with 12 reusable skills.
- A portfolio site that explains each skill as an "agent."
- A LinkedIn post that tells the story.
- A startup scorecard for tools claiming to help your job.
- A founder feedback message template.
- A scary-tool review template.
- A tactical course notes template.

## How To Use This Repo

Start here:

1. Fill out `templates/day-tracker-worksheet.md`.
2. Use `templates/skill-readme-template.md` to create your own 12 skills.
3. Review the example skills in `skills/`.
4. Give `site/lovable-v0-portfolio-prompt.md` to Lovable, v0, Claude Design, or another site builder.
5. Adapt `social/linkedin-launch-post.md` and post it with your site and GitHub links.

## Safety Rules

Do not publish private data.

Keep out:

- Customer names
- Borrower data
- Employee-sensitive notes
- Internal strategy
- Passwords
- API keys
- Private reports
- Screenshots with personal information

Publish patterns, workflows, and lessons. Do not publish confidential details.

## Weekly Cadence

Week 1: Track the real workday and choose the 12 skill ideas.

Week 2: Build the skills and publish the GitHub repo.

Week 3: Build the portfolio site and post the launch.

Week 4: Research startups, message founders, try one scary tool, and publish the follow-up.

Repeat monthly.

## Proof Of Completion

You finish the challenge when you have:

- One public GitHub repo with 12 skills.
- One portfolio site with 12 agent cards.
- One LinkedIn post linking to the site and repo.
- One startup/tool scorecard.
- One founder message sent with real feedback.
- One lesson from a tactical AI course turned into a new skill.

## Default Voice

Plain English. Specific examples. No hype.

Write like an operator showing the work, not a guru selling a course.
